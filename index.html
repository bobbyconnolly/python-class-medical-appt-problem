Final Exam Practice Problem: Medical Appointment System

Problem Statement

You are developing a medical clinic's appointment system. Create a Python class called MedicalAppointment that manages individual appointments. The class should handle appointment scheduling, status tracking, and basic validation.

Exam Tips:
  • Write your code clearly and neatly on paper
  • Leave space between methods for potential corrections
  • Draw a line through mistakes instead of erasing
  • Use consistent indentation (preferably 4 spaces)

Implementation Steps

Step 1: Plan Your Class

Before writing any code, list the attributes and methods you'll need:

Required Attributes:
  • Patient name
  • Start time (datetime)
  • Duration in minutes
  • Status (scheduled, completed, cancelled)
  • Notes list
Required Methods:
  • Constructor
  • Properties for validation
  • Appointment management methods
  • String representation
Planning helps you organize your thoughts and ensures you don't forget important components. It's also helpful during the exam to have a checklist of what you need to implement.

Step 2: Write the Constructor

Start with the __init__ method to set up your object's initial state.

Tasks:
  1. Define constructor parameters:
    • patient_name (required)
    • start_time (required)
    • duration_mins (optional, default to 30)
  2. Initialize attributes with appropriate default values
  3. Use underscore prefix for attributes that will have property decorators
We start with the constructor because it defines the core data structure of our class. Every other method will work with these attributes, so it's important to establish them first.

Step 3: Add Property Decorators

Implement properties for attributes that need validation.

Tasks:
  1. Create a property for patient_name:
    • Validate it's a non-empty string
    • Strip any leading/trailing whitespace
  2. Create a property for duration_mins:
    • Ensure it's a positive integer
    • Raise ValueError if invalid
We add properties next because they ensure our data stays valid throughout the object's lifecycle. This prevents invalid states that could cause problems in other methods.

Step 4: Implement Core Methods

Add methods that handle basic appointment operations.

Tasks:
  1. Implement get_end_time():
    • Calculate end time using start_time and duration
    • Return a datetime object
  2. Implement cancel_appointment():
    • Check if appointment can be cancelled
    • Update status appropriately
  3. Implement add_note(note_text):
    • Add timestamped note to notes list
    • Validate note is not empty
Core methods come next because they build on our validated attributes to implement the class's main functionality.

Step 5: Add Helper Methods

Implement utility methods that make the class more useful.

Tasks:
  1. Implement conflicts_with(other_appointment):
    • Check for time overlap with another appointment
    • Return True if appointments conflict
  2. Implement __str__():
    • Return formatted string with appointment details
    • Include patient name, time, duration, and status
Helper methods come last because they often combine core functionality in useful ways and make the class more practical to use.

Testing Your Implementation

While you can't run the code during the exam, you should think through these scenarios:

  1. Creating an appointment with valid data (should work)
  2. Setting invalid patient name (should raise ValueError)
  3. Setting negative duration (should raise ValueError)
  4. Cancelling a completed appointment (should raise ValueError)
  5. Checking overlapping appointments (should detect conflict)
Remember: You can write small code snippets showing how your class would be used - this helps verify your implementation makes sense!
